# 水匠5 評価関数（NNUE）— 解説と使い方

このリポジトリで取り出した `suisho5_nn/nn.bin`（水匠5の評価関数）について、
**仕組み・正当性の検証・最新やねうら王/8.60との組み合わせ方・推奨設定**をまとめた学習ノートです。

---

## 1. 評価関数とは

将棋ソフトは「指し手を読む**探索部**」と「局面の良し悪しを数値化する**評価関数**」でできています。
評価関数は

> 盤面（Position）を入力 → 「手番側がどれくらい有利か」を1つの整数（評価値）で出力

する関数です。水匠5はこの評価関数に **NNUE**（Efficiently Updatable Neural Network）を採用しています。

---

## 2. 水匠5の評価関数の正体

`nn.bin` の先頭ヘッダを解析すると、構造を表す文字列が埋め込まれています：

```
version      = 0x7AF32F16
hash         = 0x3E5AA6EE         ← ネットワーク構造ハッシュ（互換性判定に使う）
architecture =
  Features = HalfKP(Friend)[125388 -> 256x2],
  Network  = AffineTransform[1<-32](
               ClippedReLU[32](
                 AffineTransform[32<-32](
                   ClippedReLU[32](
                     AffineTransform[32<-512](
                       InputSlice[512(0:512)])))))
```

これは NNUE の最も有名な構成、通称 **HalfKP 256x2-32-32** です。

```
盤面
 │ ① 入力特徴量 HalfKP（125388次元の 0/1 ベクトル、ただし同時に1になるのは最大38個）
 ▼
[FeatureTransformer] 125388 → 256   ★評価関数ファイルの99%を占める巨大テーブル
 │ 自分視点256 + 相手視点256 = 512
 ▼
[ClippedReLU] → [Affine] 512→32 → [ClippedReLU] → [Affine] 32→32 → [ClippedReLU] → [Affine] 32→1
 ▼
評価値（÷ FV_SCALE(=24) して返す）
```

### 入力特徴量 HalfKP
- 「**自玉の位置** × **玉以外の駒の位置・種類**」の組み合わせを特徴にする
- 次元数 = `SQ_NB(81) × fe_end(1548) = 125388`（ヘッダの数字と一致）
- 盤上の玉以外の駒は最大38個 → **12.5万次元のうち常に最大38個だけが1**（残りは全部0）

### なぜ速い？（NNUE の N = 効率的に更新可能）
- 中間結果（256次元×2視点）を `Accumulator` に保持
- 1手指すと変化する特徴はごく一部 → **差分更新**だけで済む（玉が動いた時だけ全計算）

### サイズの内訳（なぜ約64MBか）
- FeatureTransformerの重み: `125388 × 256 × 2byte(int16) ≈ 64.2 MB` ← **ファイルのほぼ全部**
- 後段の3層(512→32→32→1)は int8 で合計わずか数十KB

## 3. 正当性の検証（公式Suisho5とバイト一致）

公式リリース [Suisho5.7z](https://github.com/yaneurao/YaneuraOu/releases/tag/suisho5)
の中身（`nn.bin` 単体）と、本リポジトリがソース埋め込みから取り出した `nn.bin` を SHA256 照合：

```
768068f0d534a0603a5d38bcd143de6bbca820d5f1c95a14d40863e5b7892d76  suisho5_nn/nn.bin（本リポジトリ抽出版）
768068f0d534a0603a5d38bcd143de6bbca820d5f1c95a14d40863e5b7892d76  公式 Suisho5.7z の nn.bin
→ byte identical（完全同一）
```

→ **本リポジトリの抽出は1バイトの狂いもない本物**。
公式リリースは評価関数(`nn.bin`)のみを配っており、**エンジン本体は別途やねうら王を用意して組み合わせる**前提になっています。

## 4. 最新やねうら王 / 8.60 との互換性

### 結論：標準NNUE(halfKP256)ビルドなら、バージョンに関係なく動く

NNUEは「評価関数ファイルの構造ハッシュ」と「エンジンがコンパイル時に決めた構造ハッシュ」が
一致しないと読み込みを拒否します（`evaluate_nnue.cpp` の `hash_value != kHashValue` 判定）。

**重要：このハッシュはネットワーク構造だけで決まり、やねうら王のバージョン番号には依存しない。**
HalfKP256 の構造は2018年頃から現在まで不変なので、

- 水匠5の `nn.bin`（halfKP256, 2021年）は
- やねうら王 **8.60** でも **最新master** でも、**標準NNUE(halfKP256)ビルドのexe**なら そのまま読める

### 実機検証済み
本リポジトリの `engine/` にある **最新やねうら王（9.30git, AVX512VNNI標準NNUE）** に
水匠5の `nn.bin` を読ませて `bench` を完走させ、互換性を実証しました（§6）。

### 落とし穴：パック内のexe選択
やねうら王のパックには複数エンジンが入っています。**標準NNUE型だけ**が水匠5を読めます。

| 実行ファイル名の例 | 水匠5(halfKP256)読込 |
|---|---|
| `YaneuraOu_NNUE-*`（標準・halfKP256） | ✅ **これを使う** |
| `YaneuraOu_NNUE_halfkpe9-*` | ❌ 構造違いで弾かれる |
| `YaneuraOu_NNUE_kp256-*` | ❌ |
| `..._halfkp_1024x2...` / `512x2...` | ❌ |
| `YaneuraOu_MATERIAL...`（駒得） | ❌ NNUE非対応 |

CPU向けには `AVX512VNNI` 版を選ぶと、Ryzen 9000系(Zen5)等のVNNIをフル活用できます。

### 使い方
1. 水匠5の `nn.bin` を、エンジンと同じフォルダ（または `EvalDir` 指定先）に置く
2. 標準NNUE(halfKP256)・AVX512VNNI版のexeを起動
3. 起動時に `info string loading eval file : .../nn.bin` が出て、Errorが出なければ成功

## 5. 推奨USIオプション（高クロック・多スレッド・大L3キャッシュ機向け）

水匠5は**小型ネットで高NPS**なのが特徴。これを活かす設定：

| オプション | 推奨 | 理由 |
|---|---|---|
| `Threads` | コア/スレッド数に合わせる（例32） | 小型ネットはスレッド増の伸びが良い |
| `USI_Hash` | hashfullを見て決める（下記） | 高NPSだと置換表をすぐ食う |
| `EvalDir` | `nn.bin` を置いたフォルダ | 水匠5評価関数を読ませる本体 |
| `EvalHash` | 大きめ | 評価値再利用でNPS実効値↑ |
| Large Page | 有効化推奨 | 巨大ハッシュ・評価関数のアクセス高速化 |

### ハッシュサイズの決め方（hashfull で機械的に判断）
探索中の `info` 行に出る `hashfull`（置換表使用率, 0〜1000）が唯一の客観指標：

| 1手の思考終了時の hashfull | 判断 |
|---|---|
| 常に 950〜1000 に張り付く | 足りない → ハッシュを増やす |
| 600〜900 | ちょうど良い |
| < 500 | 余っている → 増やしても効果ほぼ無し |

注意：
- **L3キャッシュとは無関係**。数GBのハッシュはL3に乗らず必ずRAMランダムアクセス。
  「大L3だから巨大ハッシュが有利」ということはない（L3が効くのは評価関数・探索側）。
- **効果は逓減**。飽和していないのに倍にしても数Elo程度。「飽和解消の1段階目」が一番効く。
- 巨大ハッシュは確保・`Clear Hash`に時間がかかる。32GB程度が実用上限の目安。

## 6. 実証ベンチ結果（最新やねうら王 × 水匠5評価関数）

`engine/YaneuraOu-NNUE-AVX512VNNI-linux`（YaneuraOu NNUE 9.30git, AVX512VNNIビルド）に
水匠5 `nn.bin` を読ませ、`bench` を実行：

```
id name YaneuraOu NNUE 9.30git 64AVX512VNNI
info string loading eval file : .../eval/nn.bin     ← 水匠5評価関数、エラーなしでロード
...
info depth 26 ... score cp -4443 ... nps 530256
Total time (ms) : 60002
Nodes searched  : 33,676,511
Nodes/second    : 561,256
```

- 評価値(`score cp`)を出し深さ26まで読破 = 評価関数として正常機能
- ※この計測機は4コア2.8GHzの非力なXeon。高クロック・多スレッド機なら**桁違いのNPS**になる
  （小型ネットの水匠5は「深さで殴る」運用が最も映える）
